How to Use This Palette
For humans and AI tools — a plain-English guide to applying the Broadlab palette correctly.
1
Primary colour
Use Teal 500 (#10657F) for all primary buttons, links, and key interactive elements. This is the dominant brand colour and should appear most frequently.
2
Navigation & chrome
Use Navy Slate (#1E4F80) for navigation bars, headers, and structural UI chrome. Never use the primary teal for navigation — keep them visually distinct.
3
Accents
Use Deep Purple (#4C2FA8) only for premium or special indicators such as badges and highlights. Use Coral Rose (#C83B2A) only for flags, alerts, and attention elements. Both are supporting colours — use sparingly.
4
Backgrounds
Use Teal 50 (#EAF6FB) for page backgrounds and large panel fills. Use Gray 50 (#F9FAFB) for card surfaces and data table rows. Never use a brand colour at full saturation as a background.
5
Text
Use Gray 900 (#111827) for primary body text. Gray 600 (#4B5563) for secondary and muted text. Gray 400 (#9CA3AF) for placeholders and hints. Never use pure black (#000000).
6
Status colours
Use Success (#16A34A) for positive indicators. Warning (#D97706) for caution states. Danger (#DC2626) for errors and off states. These are semantic — do not repurpose them for decorative use.
7
Charts
Follow the chart palette sequence in order. Start with Teal 500, then Navy, then Purple, then Coral. Never use semantic colours (success/warning/danger) as chart series colours unless they encode that specific meaning.
8
Typography
Use Plus Jakarta Sans Bold for all headings (H1, H2, H3). Use Inter Regular for body copy and UI text. Use Inter Light for captions and metadata. Both available free at fonts.google.com.
